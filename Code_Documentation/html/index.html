DECOMPOSITION AND MERGING OF A POLYGON INTO CONVEX POLYGONS

DECOMPOSITION ALGORITHM :

Procedure Description Involves storing the vertices of the next future convex polygon in a list L and generating the convex polygon by adding adjacent vertices of P in clockwise order while satisfying the given certain conditions. The final convex polygon is obtained by drawing a diagonal joining the last and first vertices of L. The algorithm stops when all the vertices of P are in L or when a vertex fails one of the two conditions. The checking of the interior vertices of P in the convex polygon is reduced to the notches found in the rectangle containing all the vertices of L. If a notch is found to be inside the polygon generated by L, it is removed from L, and the process is repeated until no notch is inside the polygon. The algorithm generates one of the polygons of the partition if L has more than two vertices, else it does not generate a polygon in this call.

MERGING ALGORITHM :

If a diagonal, d, is essential for a vertex, v, then removing d will result in a non-convex piece at v. A diagonal that is not essential for any vertex is called inessential. The algorithms presented above produce partitions that may sometimes contain inessential diagonals. To prevent this, we have developed a merging process that can be used after any partitioning process producing inessential diagonals. Each diagonal of the partition is considered in order, and it is checked whether it can be removed. If the polygons, Ps and Pt, sharing a given diagonal, have already been merged to other polygons, P's and P't, then the polygons to be considered when checking whether that diagonal can be removed are the merged polygons, Ps U P's and Pt U P't, and not the single polygons, Ps and Pt, sharing the diagonal
